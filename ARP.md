**ARP(Address Resolution Protocol)** <br>
ARP란 그대로 해석 하면 주소 결정 프로토콜이라는 뜻이며, 일반적으로 통신은 사용자가 어떤 값을 생성하여 전달하거나 어떤 정보를 요청하기 위해 동작을 시
킬때 발생됩니다. 즉, 통신을 위해 필요한 일련의 값들을 사용자가 결정하여 입력을 시켜준다는 뜻입니다. 4계층에서 필요한 서비스 구분번호는 어플리케이션
을 동작시키면서 입력이 되고, 3계층에서 사용하는 IP주소 까지 사용자가 통신하기 위해서 디바이스에 입력하기 때문에 어렵지 않게 Encapsulation이 가능합
니다. 하지만 2계층에서 필요한 목적지 주소는 따로 입력이 되지 않아서 주소를 디바이스 스스로 결정해주어야 합니다. 이때 사용하는 프로토콜이 ARP입니다. 
정리를 해보면, ARP란 논리적인 IP주소를 기반으로 (데이터링크 계층의) 물리적인 MAC주소로 바꾸어 주는 주소해석 프로토콜입니다. <br>
2계층에서 물리적 주소를 사용하여 통신하는 경우 목적지 주소를 알아와서 Encapsulation을 완성하기 위한 목적으로 쓰입니다. <br>
IP를 이용하여 MAC주소를 알아내기 위해 연결된 모든 장비에게 Request 메세지를 보내고, ARP Request 메세지를 받은 장비들은 누구에게 전달된 Request인지 
IP주소를 확인하여 자신의 IP일 경우 Reply를 보내줍니다. <br>
ARP프로토콜은 Request를 보내기 위해 브로드캐스트 방식을 사용합니다. <br>
목적지 IP주소가 같은 네트워크일 경우에는 해당 장비를 직접 찾는 Request 메세지를 브로드캐스트로 보내주고, 다른 네트워크일 경우에는 직접 브로드캐스트
를 보낼수 없으니 Gate-way로 데이터를 전달하기 위해 GW에게 Request를 보내줍니다. <br>
즉, 브로드캐스트가 도달할 수 없는 다른 네트워크에는 ARP를 직접 보낼 수 없습니다.

-----
*ARP Cache Table* <br>
ARP로 MAC주소를 알아오는 과정에서 브로드캐스트 방식을 사용하는데, 브로드캐스트는 많이 일어날 수록 장비의 과부하와 성능저하를 가져오기 때문에 좋지 
못합니다. 그래서 한번 알아온 정보는 ARP Table에 일정시간 동안 저장하고, 통신하고자 하는 목적지의 MAC주소가 ARP Table에 저장되어 있는 경우 ARP 
프로토콜이 동작하지 않습니다. <br>
`arp -a` 라는 명령어를 사용하여 저장되어 있는 ARP Mapping 정보를 확인할 수 있고, ARP 프로토콜을 사용하여 알아온 동적인 정보는 Dynamic(동적) 이라고 
표시됩니다. 브로드캐스트의 단점을 보완하기 위해 관리자가 직접 명령어를 이용하여 수동 입력할 수 있고, 그런 경우에는 Static(수동) 이라고 표시됩니다.

-----
*ARP의 종류* <br>
1. ARP : 위의 설명이 있기 때문에 생략.
2. RARP : RARP는 ARP의 반대 개념으로 MAC주소로 IP주소를 가져오는 것을 말합니다. 일반적으로 IP주소는 하드디스크 내의 구성파일에 의해 설정되지만, 
하드 디스크가 없는 X 터미널이나 워크스테이션의 경우, 초기 가동될 때 자신의 MAC주소를 담아 RARP Request를 보내서 IP 주소 정보를 받아오기 위해서 
사용 되었습니다. <br>
3. Proxy ARP : 라우터가 ARP 프레임을 수신했을 때, 해당 네트워크가 자신의 라우팅 테이블에 있으면 라우터 자신의 MAC주소를 알려주는 기능을 말합니다.
4. Inverse ARP : Frame-Relay 환경에서 송신지 IP와 목적지 DLIC를 맵핑하여 FR MAP에 저장하는데, 처음 연결을 한 경우 해당 목적지 IP에 대한 정보가 
없으므로 자신과 연결된 모든 선로로 멀티플렉싱을 해서 목적지 IP를 알아내는 것을 말합니다. <br>
5. Gratuituous ARP : 같은 네트워크에서 IP주소가 중복되는지 확인하고 다른 장치들이 자신의 IP주소를 갱신하도록 주기적으로 IP주소를 알려주는 역할을 
합니다.
